                        DATABASE SCHEMA (13 MODELS)
═══════════════════════════════════════════════════════════════════════════════

1. User (Authentication & Profiles)
   Fields: id, email, phone, passwordHash, firstName, lastName, role, status,
           avatar, bio, region, emailVerified, phoneVerified, timestamps

2. Vendor (Vendor Profiles)
   Fields: userId, businessName, businessType, description, website,
           kycStatus, kycDocument, rating, commissionRate, earnings, payouts

3. Customer (Customer Data)
   Fields: userId, totalSpent, totalOrders, loyaltyPoints, preferences

4. Product (Marketplace Items)
   Fields: vendorId, title, description, sku, category, type (SALE/RENTAL),
           price, rentalRates (daily/weekly/monthly), images, quantity, status

5. Order (Purchase & Rental Orders)
   Fields: userId, vendorId, type (PURCHASE/RENTAL), status, subtotal,
           tax, total, shippingAddress, trackingNumber, timestamps

6. OrderItem (Line Items)
   Fields: orderId, productId, quantity, price, subtotal

7. Cart (Shopping Cart)
   Fields: userId, subtotal, tax, total, updatedAt

8. CartItem (Cart Line Items)
   Fields: cartId, productId, quantity, price

9. Payment (Payment Processing)
   Fields: orderId, userId, amount, status, method, transactionId, timestamps

10. Payout (Vendor Payouts)
    Fields: vendorId, amount, status, bankDetails, processedAt, timestamps

11. Wishlist (Saved Items)
    Fields: userId, productId, createdAt

12. Review (Product Ratings)
    Fields: productId, userId, rating, comment, verified, timestamps

13. Subscription (Plans)
    Fields: userId, vendorId, planType, status, startDate, endDate, price

Additional Models:
  ✅ RentalCalendar - Booking availability tracking
  ✅ Verification - KYC document verification
  ✅ AuditLog - Admin action tracking
  ✅ Notification - User notifications

                        API ENDPOINTS (100+)
═══════════════════════════════════════════════════════════════════════════════

Authentication Routes (/api/auth/*)
  POST /login            - User login
  POST /register         - User registration
  POST /refresh          - Refresh JWT token
  POST /logout           - User logout
  GET  /me               - Get current user profile

Product Routes (/api/products/*)
  GET  /                 - List all products
  GET  /search?q=        - Search products
  GET  /:id              - Get product details
  GET  /category/:cat    - Get by category
  GET  /wishlist/items   - Get wishlist (protected)
  POST /wishlist/add     - Add to wishlist (protected)
  DELETE /wishlist/:id   - Remove from wishlist (protected)

Cart Routes (/api/cart/*)
  GET  /                 - Get cart (protected)
  GET  /total            - Get cart totals (protected)
  POST /add              - Add item (protected)
  PUT  /items/:id        - Update item (protected)
  DELETE /items/:id      - Remove item (protected)
  DELETE /               - Clear cart (protected)

Order Routes (/api/orders/*)
  POST /                 - Create order (protected)
  POST /rental           - Create rental (protected)
  GET  /                 - Get orders (protected)
  GET  /:id              - Get order details (protected)
  POST /:id/cancel       - Cancel order (protected)

Vendor Routes (/api/vendor/*)
  GET  /profile          - Vendor profile (vendor)
  GET  /products         - Vendor products (vendor)
  POST /products         - Create product (vendor)
  PUT  /products/:id     - Update product (vendor)
  DELETE /products/:id   - Delete product (vendor)
  GET  /orders           - Vendor orders (vendor)
  GET  /earnings         - Vendor earnings (vendor)

Admin Routes (/api/admin/*)
  GET  /users            - List users (admin)
  PUT  /users/:id        - Update user (admin)
  GET  /vendors          - List vendors (admin)
  POST /vendors/:id/approve-kyc - Approve vendor (admin)
  GET  /orders           - List all orders (admin)
  GET  /stats            - Platform statistics (admin)
  GET  /finance/payouts  - Payout management (admin)
  GET  /kyc              - KYC queue (admin)
  POST /kyc/:id/approve  - Approve verification (admin)
  POST /kyc/:id/reject   - Reject verification (admin)

Health & Utility
  GET /health            - Health check
