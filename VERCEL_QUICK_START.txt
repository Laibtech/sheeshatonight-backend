🚀 8 STEPS TO DEPLOY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

STEP 1: Push Backend to GitHub (5 min)
  cd backend
  git add .
  git commit -m "Add Vercel config"
  git push origin main
  
  If repo doesn't exist:
  → Go to: https://github.com/new
  → Create: sheeshatonight-backend
  → Then: git push -u origin main

STEP 2: Create Database (15 min)
  → https://supabase.com
  → Sign up with GitHub
  → Create project: "sheeshatonight-prod"
  → Save connection string
  
STEP 3: Generate JWT Secret (1 min)
  node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"
  → Save the output

STEP 4: Deploy Backend (15 min)
  → https://vercel.com/signup
  → Sign up with GitHub
  → Import: sheeshatonight-backend
  → Add 5 env variables:
     DATABASE_URL
     JWT_SECRET
     PORT=5000
     NODE_ENV=production
     FRONTEND_URL=http://localhost:3002
  → Deploy
  → Save backend URL

STEP 5: Run Migrations (5 min)
  cd backend
  $env:DATABASE_URL="your-supabase-url"
  npx prisma migrate deploy
  npx prisma generate
  node src/utils/seed.js

STEP 6: Deploy Frontend (15 min)
  → Vercel Dashboard
  → Import: sheeshatonight-main
  → Add 3 env variables:
     NEXT_PUBLIC_API_URL=your-backend-url
     JWT_SECRET=same-as-backend
     NODE_ENV=production
  → Deploy
  → Save frontend URL

STEP 7: Update Backend CORS (10 min)
  → Vercel → Backend → Settings → Env Vars
  → Update FRONTEND_URL to your frontend URL
  → Add ALLOWED_ORIGINS with frontend URL
  → Redeploy

STEP 8: Test Everything (10 min)
  ✓ Visit backend/health
  ✓ Visit frontend homepage
  ✓ Test login
  ✓ Check browser console for errors

Database Connection Fails:
  → Check DATABASE_URL in Vercel
  → Verify password in connection string

API Calls Fail:
  → Check NEXT_PUBLIC_API_URL in frontend
  → Must be full URL: https://xxx.vercel.app

Build Fails:
  → Check Vercel build logs
  → Test locally: npm run build
